# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in KeelMatrix.Redaction, report it privately by either:

1. emailing **keelmatrix@gmail.com**
2. opening a private GitHub security advisory

Do **not** create a public issue for security vulnerabilities.

Useful report details:

- affected package version
- target framework and runtime
- reproduction steps or proof of concept
- impact and any suggested mitigation, if known

Reports are handled best-effort. Valid issues will be investigated and fixed as appropriate for the current maintained release line.

## Supported Versions

Security fixes are prioritized for the latest released package line. Older versions may receive fixes on a case-by-case basis.

## Guidelines

- Do not post details of the security issue in public repositories or issue trackers.
- Include as much information in your report as possible to help reproduce and resolve the issue.
- Additional information or a proof of concept may be requested to verify and fix the vulnerability.
